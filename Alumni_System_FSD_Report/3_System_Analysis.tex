\chapter{SYSTEM ANALYSIS }
 
\section{Module Description}
The Alumni Management System is divided into multiple functional modules to ensure a structured and scalable application. The main modules include:
\begin{itemize}
    \item \textbf{Authentication \& Authorization Module:} Handles user registration, login, and secure session management. Utilizing Laravel Breeze, this module ensures secure password hashing and role-based access control (Admin, Alumni, Student, Department).
    \item \textbf{Profile Management Module:} Allows users to update their personal and professional details. Users must complete their profile before accessing the core features of the system.
    \item \textbf{Post \& Comment Module:} Acts as the primary social feed. Users can create posts, upload media (images/videos), like posts, and engage in threaded discussions through comments.
    \item \textbf{Private Messaging Module:} Enables users to send direct messages to one another. This module ensures privacy and secure peer-to-peer communication.
    \item \textbf{Alumni Search Directory Module:} Provides an advanced search interface for finding alumni by various filters such as graduation year, department, and current profession.
    \item \textbf{Admin & Moderation Module:} Dedicated dashboards for platform administrators to approve new alumni registrations, manage user accounts, and oversee the platform's overall health and content.
    \item \textbf{Department Module:} Specialized dashboards for department heads to view and manage the alumni network specific to their faculty.
\end{itemize}

\section{Business Rules}

Business rules define how processes and activities should be conducted to ensure consistency across the application.
\begin{itemize}
    \item \textbf{Profile Completeness:} Users cannot actively participate in writing posts or sending messages until their core profile fields (such as graduation year and current job) are filled out. This ensures data integrity within the platform.
    \item \textbf{Role Permissions:} Only Administrators can approve or reject new alumni registrations. Regular students and alumni cannot perform administrative actions across the network.
    \item \textbf{Alumni Independence:} Alumni data belongs to the alumni. The platform cannot publicly expose their private email addresses; instead, peer-to-peer communication must flow through the internal messaging system.
\end{itemize}

\section{Analysis of Architecture}

\subsection{Laravel Framework (PHP 8.2)}
The Alumni Management System is built natively on the Laravel 12.x PHP framework, utilizing an MVC architecture to guarantee clean, scalable, and secure application code.
\begin{itemize}
    \item \textbf{Backend Architecture (MVC):} The Model layer handles relational PostgreSQL queries preventing SQL injection. The Controller isolates business logic (e.g., verifying user profile states), and the Blade View layer handles dynamic DOM rendering.
    \item \textbf{Frontend Utility Styling:} Tailwind CSS serves as a robust utility engine preventing cross-component cascading errors and vastly reducing custom CSS payload sizes.
\end{itemize}

\subsection{Key Algorithms}

\textbf{Profile Completion Check Algorithm:}
To maintain a high quality of data within the alumni network, a custom Middleware algorithm evaluates the user's Eloquent Profile model upon every critical HTTP request. The algorithm checks if essential columns (graduation year, major) are strictly strictly `!== null`. If any are unset, it forcefully intercepts the request and redirects the user to the profile configuration form, regardless of their intended target route. \\

\textbf{Alumni Search \& Filter Algorithm:}
The alumni directory employs an advanced dynamic query construction algorithm. Based on optional GET parameters (e.g., specific graduating class, department identifier, or job title keyword), the Controller appends conditional `where()` clauses sequentially to the central base Eloquent query before invoking the `paginate()` termination method. This ensures minimal database load by only targeting explicitly requested indices.

\section{Feasibility Analysis}
 
\subsection{Technical Feasibility}
The system is built on a modern, robust technology stack. The backend utilizes the Laravel PHP framework, known for its elegant syntax, built-in security features, and powerful Eloquent ORM. The frontend leverages Laravel Blade templating engine coupled with Tailwind CSS, ensuring a responsive and modern user interface. Vite is used as the frontend build tool for rapid development. The data is managed using a PostgreSQL database, providing a reliable and scalable storage solution for relational data. Given the well-documented nature of these technologies, the project is highly technically feasible.

\subsection{Economical Feasibility}
The development and deployment of the Alumni Management System are highly cost-effective. The chosen technologies (PHP, Laravel, PostgreSQL, Tailwind) are open-source and free to use. Hosting can be achieved on affordable cloud platforms or dedicated VPS servers without incurring massive costs. Maintenance and future scaling are straightforward due to Laravel's popularity and community support.

\subsection{Operational Feasibility}
The system is designed with a user-centric approach. The clean, intuitive interface makes it simple for non-technical users to navigate. Automated workflows, such as profile completion checks and notification alerts, guide the user seamlessly through the platform. For administrators, the dedicated dashboards significantly reduce the manual effort required to manage an alumni network.

\section{Actors and Roles}
The system involves four primary actors: \\

\noindent \textbf{Admin}
\begin{itemize}
    \item Approves or rejects new alumni registrations.
    \item Manages global platform settings and content moderation.
\end{itemize}
\noindent \textbf{Alumni}
\begin{itemize}
    \item Can post updates, search the directory, and message other users.
    \item Acts as the core user base for networking and mentorship.
\end{itemize}
\noindent \textbf{Student}
\begin{itemize}
    \item Engages with alumni posts and seeks mentorship through private messages.
\end{itemize}
\noindent \textbf{Department}
\begin{itemize}
    \item Views metrics and manages alumni specific to their department.
\end{itemize}
